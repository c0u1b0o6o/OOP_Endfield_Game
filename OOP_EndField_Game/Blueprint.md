# 專案開發藍圖：明日方舟終末地 - 源石電路修復解謎 (C++ / SFML)

## 🎯 專案總覽 (Project Overview)
本專案旨在復刻遊戲《明日方舟：終末地》中的「源石電路修復」解謎小遊戲。
玩家需要將多種不同形狀、顏色的零件，放置於 M x N 的盤面上。
目標是讓盤面每一列、每一欄的「各顏色」格子總數，完全符合盤面周圍提示的數字。

## 🛠 技術棧與架構約定 (Tech Stack & Rules)
* **語言與標準**：C++ (推薦 C++20 標準)。
* **圖形框架**：SFML 3.0.2。
* **開發守則**：
    1.  **模組化設計 (MVC)**：核心邏輯 (Core Logic) 必須與圖形渲染 (Rendering) 完全分離。
    2.  **動態讀檔 (致命規定)**：嚴禁 Hardcode 關卡設定檔路徑。必須支援在程式執行時 (Runtime) 透過命令列參數或 UI 輸入來選擇載入的檔案。若寫死路徑將直接 0 分。
    3.  **分階段實作**：請 AI 嚴格按照下方的 Phase 順序進行開發，未經指示請勿跳躍至下一個 Phase。

## 🚀 實作階段拆解 (Development Phases)

### Phase 1: 核心資料結構與邏輯 (Core Logic)
> **任務目標**：建立純粹 C++ 遊戲邏輯，支援多種顏色 (Multi-color) 的盤面狀態與驗證。

1.  **`Part` (零件類別)**：
    * 屬性：`id` (依輸入順序)、`colorIndex`、`width`、`height`、`shape` (0/1 二維陣列)。
    * 方法：`rotateRight()`。旋轉後需重新計算寬高與陣列。
    * **特殊防呆 (旋轉中心)**：在初始化時，必須動態遍歷 `shape`，找到「第一個值為 1 的實體格子」作為旋轉與滑鼠拖曳的中心點 (Pivot)。Pivot 絕對不能是 0 (空格)。
2.  **`Board` (盤面類別)**：
    * 屬性：列數 `M`、欄數 `N`、顏色數 `C`。
    * 屬性：`targetRowCounts[color][row]` 與 `targetColCounts[color][col]`。
    * 屬性：二維網格狀態 `grid`。需能區別：空格、不可放置格 (-2)、預先固定的鎖定格 (需記錄其顏色，不可移動但算作填滿 1 格)、玩家放置的零件編號。
    * 方法：`canPlace(part, row, col)`，檢查越界、重疊、是否踩到不可放置格 (-2) 或固定格。
    * 方法：`placePart(...)`、`removePart(...)`。
    * 方法：`checkWinCondition()`，驗證當前盤面各顏色於各列/欄的實體格子數(含固定格)，是否與目標數字完全一致。

### Phase 2: 關卡載入器 (Level Parser)
> **任務目標**：精準解析規定的 `.txt` 設定檔，支援字串緊湊格式。

1.  **檔案格式解析規則 (嚴格遵守順序)**：
    * 讀取 `C M N`。
    * Loop `C` 次讀取各顏色的：`M` 個列目標、`N` 個欄目標。
    * 接著讀取該顏色固定格數量 `X1`。接下來 `X1` 行為 `row col` 座標 (寫入 Board 作為該顏色的固定格)。
    * Loop 結束後，讀取不可放置格數量 `X2` 與其座標 (寫入 Board 狀態為 -2)。
    * 讀取零件列表直到 EOF：`ColorIndex M2 N2`。
    * **注意陷阱**：接下來的 `M2` 行，每行是連續的字串 (例如 `110`)，請以字元 (char) 逐一解析轉換為 1 或 0 存入陣列，不要當作以空白分隔的整數讀取。

### Phase 3: SFML 圖形引擎與進階 UI (GUI & Input)
> **任務目標**：實作遊戲迴圈，包含動畫、音效與各項進階提示。

1.  **渲染與 UI (Rendering & UI)**：
    * 實作精美的主畫面 (Main Menu) 與關卡選擇介面。支援不關閉程式直接開新局。
    * 繪製盤面網格，自訂材質區分：不可放置格(灰)、固定格(帶鎖頭標籤)、不同顏色的玩家零件。
    * **動態提示**：盤面周圍繪製列/欄的目標數字。當玩家放置零件後，若該列/欄的填滿數「等於」或「超過」目標數，數字必須改變顏色進行視覺提示。
2.  **操作邏輯 (Input)**：
    * 支援 W/A/S/D 或滑鼠平滑拖曳來移動選取的零件。
    * 拖曳中按特定鍵旋轉零件，必須包含「旋轉動畫」(Smooth Rotation Animation)。
    * 左鍵/Enter 嘗試放置。若違反規則 (如重疊)，印出錯誤訊息並播放失敗音效；成功則播放放置音效。
    * 支援按下按鈕「重置盤面」(Reset Board) 回到初始讀檔狀態。

### Phase 4: 自動解題機與提示系統 (Auto-Solver & Hint)
> **任務目標**：利用 DFS + Backtracking 在背景找解答，並支援標準輸出與 GUI 提示。

1.  **演算法實作**：
    * `class AutoSolver` 必須具備「剪枝 (Pruning)」功能。放置前先檢查當前行列數量是否已爆表，提早回溯。
    * **終端機輸出要求**：當找到解答時，能在 Console 印出純文字解答：固定零件印 `=`，不可放零件格印 `X`，空格印 ` `，有放零件印其 ID。
2.  **GUI 提示整合**：
    * 若玩家閒置超過 30 秒卡關，開放提示按鈕。
    * 按下後，抓取 Solver 解答的下一步，在盤面上以「半透明綠色區塊」顯示該零件的正確位置與形狀。

### Phase 5: 關卡設計器 (Level Editor)
> **任務目標**：遊戲內自製關卡並匯出 `txt`。

1.  **編輯器功能**：
    * 允許設定 `M`、`N` 與顏色數 `C`。
    * 滑鼠點擊盤面設定不可放置格、各顏色固定格、修改周圍提示數字。
    * 零件創造器：指定顏色與大小，透過點擊網格繪製「任意形狀」的零件。
2.  **試玩與匯出**：
    * 包含 `Export` 按鈕，將自創關卡嚴格按照 Phase 2 的 `.txt` 格式輸出。
    * 包含 `Test Play` 按鈕，直接載入當前盤面進入 Phase 3 的遊戲模式。
